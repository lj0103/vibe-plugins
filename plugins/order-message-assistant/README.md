# 订单消息助手

用于快递助手 ERP 与抖店飞鸽客服工作台之间的订单消息自动化。当前规则只处理蓝旗订单：发送换货说明，确认成功后将订单备注旗子改为灰旗。

## 功能

- 自动识别快递助手当前筛选结果中的订单。
- 仅处理蓝旗订单，无旗及其他颜色旗子全部跳过。
- 向对应买家发送固定的“换成紫色发货”说明。
- 消息确认发送成功后，将蓝旗修改为灰旗。
- 自动滚动虚拟订单列表，并用稳定订单指纹对重叠批次去重。
- 日志同时显示本地订单键、昵称、收件人及脱敏地址摘要。
- 支持演练模式、停止、放弃当前任务和清除本地进度。

## 环境要求

- Chrome 或 Chromium 内核浏览器
- Tampermonkey
- 已登录的快递助手 ERP 和抖店飞鸽客服工作台
- Chrome 开发者模式（用于加载配套扩展）

## 安装

1. 按照 [extension/安装说明.md](./extension/安装说明.md) 加载“订单消息助手·飞鸽真实输入”扩展。
2. 在 Tampermonkey 中新建脚本。
3. 将 [src/order_message_assistant.user.js](./src/order_message_assistant.user.js) 的完整内容粘贴并保存。
4. 刷新快递助手 ERP 和飞鸽客服页面。

## 首次校准

在快递助手订单页依次校准：

1. `一整条订单记录`
2. `进入订单/联系买家入口`
3. `已有旗子时的备注铅笔`
4. `已有任意旗子标志`
5. 手动打开一次备注弹窗
6. `弹窗中的灰旗选项`
7. `备注保存按钮`

校准点击只记录控件，不执行对应操作。页面结构变化后可使用“重置校准”重新记录。

## 使用

1. 在 ERP 中先完成订单筛选。
2. 使用演练模式检查少量订单的识别和跳过结果。
3. 确认蓝旗识别、订单定位和日志正确后，切换到正式模式。
4. 点击“开始”。脚本会逐批扫描当前筛选结果，只处理蓝旗订单。

## 安全机制

- ERP 和飞鸽页面通过订单号、买家会话和稳定行指纹交叉校验。
- 找不到唯一订单行、目标会话、输入框或旗子控件时立即停止。
- 每次只允许一个 ERP 页面作为任务控制者，避免多个标签页争抢任务。
- 飞鸽输入由配套扩展通过浏览器级真实输入完成，并在发送后检查输入框或消息气泡。
- 已完成指纹仅保存在 Tampermonkey 本地存储中；“清除进度”不会修改平台订单。

## 修改消息

编辑用户脚本顶部的 `BLUE_FLAG_MESSAGE`。扩展会接收主脚本本次传入的消息，无需同步修改扩展源码。消息长度上限为 800 字。

## 开发与检查

项目没有额外构建步骤，可直接进行语法检查：

```bash
node --check src/order_message_assistant.user.js
node --check extension/background.js
node --check extension/content.js
```
